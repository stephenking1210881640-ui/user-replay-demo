# 产品术语表

本文档用于统一用户回放平台的页面文案、组件文案和后续需求实现中的产品表达。

## 核心对象

| 推荐术语 | 使用场景 | 避免混用 |
| --- | --- | --- |
| 旅程 | 平台核心对象，指一次用户行为链路及其分析数据 | 样本、记录、回放记录 |
| 研究项目 | 旅程归档与分析结论沉淀的正式对象 | 项目、研究任务 |
| 用户 | 平台中的行为主体 | 访客、客户，除非页面明确区分身份类型 |
| 应用 | 租户下接入平台的数据来源 | 站点、系统，除非描述 host 或业务系统 |
| 标签 | 用户标签和旅程标签的统称 | 标记、分类 |

## AI 文案

| 推荐术语 | 使用场景 |
| --- | --- |
| AI 摘要 | 列表、表格、卡片等需要短句概括的区域 |
| AI 总结 | 详情页中固定结构的完整分析区域 |
| 一句话总结 | AI 总结中的短摘要字段 |
| 目标达成分析 | 判断旅程目标是否完成及阻断点 |
| 异常行为摘要 | 说明异常、错误反馈、长停留或阻断信号 |

## 旅程状态

| 数据状态 | 页面展示 |
| --- | --- |
| `COMPLETED` | 已完成 |
| `ABANDONED` | 未完成 |
| `FAILED` | 异常 |
| `BROWSING` | 仅浏览退出 |

## 文案规范

- 列表页优先使用短句，突出可扫描性，例如“最近旅程 AI 摘要”“异常旅程占比”。
- 详情页可以使用完整说明，但每段描述控制在一句话内，优先回答“发生了什么、为什么重要、下一步做什么”。
- CTA 使用动词开头，例如“查看异常旅程”“加入研究项目”“返回研究项目列表”。
- 空状态应说明当前缺少什么，并给出可执行入口，例如“当前研究项目还没有旅程，请先从旅程列表选择代表性旅程”。
- 涉及研究项目时使用完整术语“研究项目”，按钮、表头和空状态中不要单独写“项目”。
